# Docs / Feature 记录

## Feature 名称
Qwen3.6 长篇小说分镜生成（10W字级别）

## 目标
输入一部约 10 万字中文小说，生成可执行的完整分镜草案（章节 -> 场景 -> 镜头）。

## 可行性结论
- `qwen3.6-plus` 的输入上下文可覆盖 10W 字小说（单次可读）。
- 单次调用难以输出“整本完整分镜”，主要受最大输出 token 限制，直接全量生成容易被截断。

## 已知约束
- 上下文窗口：`1,000,000 tokens`
- 最大输出：`65,536 tokens`
- 风险：长输出截断、前后设定漂移、人物/道具一致性下降

## 推荐实现方案（三段式）
1. 全书结构抽取  
   输出人物表、地点表、时间线、章节摘要、剧情主线。
2. 分章/分场景分批生成分镜  
   以章节或场景为单位调用模型，统一输出结构化字段：`scene_id`、`shot_id`、镜头类型、景别、时长、画面描述、对白、音效、转场、角色出场。
3. 全局一致性校对与合并  
   校正人设、道具、时间线冲突，汇总为完整分镜文档。

## 最小交付标准（MVP）
- 覆盖率：100% 章节均有分镜
- 截断率：0（每批结果可完整返回）
- 一致性：关键角色名、地点名、核心事件无冲突
- 可导出：`Markdown + JSON`

## 待实现接口建议
- `POST /api/storyboard/novel/parse`
- `POST /api/storyboard/novel/generate-batch`
- `POST /api/storyboard/novel/merge-and-validate`
- `GET /api/storyboard/novel/{taskId}`

## 备注
本 feature 的核心不是“单次生成全部结果”，而是“单次读全书 + 多批次稳定产出 + 最终一致性合并”。
