# JianDou 前端功能文档

更新时间：2026-04-16

本文档基于当前前端代码实际接入情况整理，目标是给其他 AI 或设计工具提供足够清晰的功能边界，用于重新生成 UI。本文只描述“前端已经对接或已经实现”的能力，不把未来规划当成现状。

## 1. 产品定位

JianDou 是一个面向内容团队的文生视频工作台，核心目标是把“小说正文 / 创意提示词”转成视频生成任务，并在同一工作台里完成：

1. 参数配置
2. 文本输入
3. AI 生成提示词
4. 创建生成任务
5. 实时查看任务进度
6. 任务列表筛选、巡检、重试、暂停、继续、终止、删除
7. 对任务效果打分，沉淀可复用 Seed

## 2. 当前信息架构

## 3. 全局壳层与导航

### 3.1 全局导航

左侧侧边栏当前包含 3 个导航项：

1. 官网首页
2. TXT 小说生成
3. 任务管理

移动端有顶部栏和侧边栏抽屉。

### 3.2 开发者模式

侧边栏下方有一个“开发者模式”区域，属于真实功能，不是纯展示：

1. 开启开发者模式
2. 开启“在关键帧前停止”
3. 设置保存到浏览器本地 `localStorage`

该设置会影响创建任务时是否传递 `stopBeforeVideoGeneration=true`。因此 UI 里应把它视为“全局工作台设置”，不是某个页面装饰。

## 4. 首页功能说明

首页本质上是营销/入口页，不承担业务操作。

### 4.1 首页模块

1. Hero 主视觉
2. 产品能力说明
3. 案例展示
4. 两个跳转 CTA：
   - 开始生成
   - 查看任务

### 4.2 首页信息属性

首页展示的是静态内容，不调用业务接口，不依赖实时数据。

## 5. 生成页功能说明

生成页是当前最核心的业务入口。页面结构是“左侧参数配置 + 右侧创作输入 + 右侧进度区”。

## 5.1 页面目标

用户在一个页面内完成：

1. 选择模型链路
2. 配置画幅、分辨率、输出数量、时长限制、Seed
3. 录入或上传正文
4. 一键生成创意提示词
5. 创建生成任务
6. 实时查看创建后任务进度和最近追踪

## 5.2 页面模块

### A. 任务框架

字段：

1. 任务标题，必填
2. 画幅比例，当前只有 `9:16` / `16:9`

### B. 模型链路

字段：

1. 文本模型，必选
2. 视觉模型，必选
3. 关键帧模型，必选
4. 视频模型，必选

说明：

1. 页面启动时会先请求生成选项目录
2. 默认不会替用户预选模型，要求用户明确选择
3. 顶部会显示已选择模型数量，满值为 4

### C. 输出控制

字段：

1. Seed，可为空，只接受非负整数
2. 清晰度 / 画幅，从视频尺寸选项里选择
3. 输出视频数量，支持 `auto` 或 1-20 条
4. 视频总时长限制，支持 `auto` 或手动
5. 手动最大总时长，仅在手动模式下出现

实际约束：

1. 视频尺寸会根据“当前画幅”和“当前视频模型支持能力”动态过滤
2. 手动时长必须满足当前模型的最小时长要求
3. 手动时长允许范围是 1-120 秒，但会再叠加模型最小时长校验

### D. 高分 Seed 复用

该模块是真实业务能力，不是展示卡片。

功能：

1. 读取任务列表，按 `effect_rating_desc` 排序
2. 仅保留“有 Seed 且有评分”的任务
3. 最多展示 8 条
4. 点击某条记录后，把该 Seed 回填到当前表单

每个候选 Seed 卡片当前展示：

1. Seed 值
2. 评分
3. 评分时间
4. 任务标题
5. 画幅 / 任务状态 / 任务 ID
6. 评分备注
7. 当前是否已回填

### E. 全局提示词

功能：

1. 手动编辑提示词
2. 点击“AI 生成提示词”
3. 清空提示词
4. 展示提示词来源

AI 生成提示词时实际传参包括：

1. 任务标题
2. 画幅比例
3. 时长
4. 正文文本
5. 源文件名
6. `editingMode = drama`

### F. 小说 TXT 输入

功能：

1. 上传 TXT 文件
2. 读取文件内容并自动填充到正文输入框
3. 同时把文件上传到后端
4. 如果标题为空或还是默认标题，则自动用文件名改写任务标题

当前 UI 里，TXT 区既支持上传，也支持直接粘贴正文。

### G. 提交区

动作：

1. 生成视频
2. 查看任务列表
3. 查看当前任务

提交前校验：

1. 标题非空
2. 四个模型全部已选
3. 时长校验通过

提交时传递的关键业务参数：

1. title
2. creativePrompt
3. aspectRatio
4. textAnalysisModel
5. visionModel
6. imageModel
7. videoModel
8. videoSize
9. outputCount
10. seed
11. `videoDurationSeconds = "auto"`
12. minDurationSeconds / maxDurationSeconds
13. transcriptText
14. stopBeforeVideoGeneration

### H. 实时进度卡

这是创建任务后的即时反馈区，当前为页面右侧固定模块。

显示内容：

1. 当前阶段名称
2. 百分比进度条
3. 任务状态
4. 任务 ID
5. Trace 数量
6. 已耗时
7. 最后更新时间
8. 如果任务已经产出视频，则显示视频预览

进度机制：

1. 创建任务成功前，先展示乐观进度
2. 拿到任务 ID 后，切换为轮询真实任务详情 + trace
3. 轮询周期约 2 秒

### I. 最近追踪

显示最近 6 条 trace 事件，按倒序展示。

每条显示：

1. 阶段
2. 消息
3. 时间
4. 级别

## 5.3 生成页状态设计

至少需要覆盖以下状态：

1. 配置加载中
2. 配置加载失败
3. 正在上传 TXT
4. 正在生成提示词
5. 正在创建任务
6. 任务运行中
7. 任务暂停
8. 任务失败
9. 任务完成
10. 高分 Seed 加载失败

## 6. 任务页功能说明

任务页是“看板 + 列表 + 详情 + 运维动作”的组合页面。

## 6.1 页面目标

在一个页面内完成：

1. 浏览全部任务
2. 搜索、筛选、排序
3. 按生命周期分组查看
4. 切换卡片视图 / 列表视图
5. 查看任务详情
6. 执行暂停、继续、终止、重试、删除
7. 对任务效果评分
8. 查看近期日志、执行监控、创建参数

## 6.2 顶部筛选区

功能：

1. 搜索任务
2. 状态过滤
3. 视图切换：列表 / 卡片
4. 排序
5. 清空筛选
6. 创建新任务入口

当前支持的排序：

1. 最近更新
2. 最新创建
3. 进度优先
4. 文本输入优先
5. 评分最高

当前支持的状态过滤：

1. PENDING
2. PAUSED
3. ANALYZING
4. PLANNING
5. RENDERING
6. COMPLETED
7. FAILED

说明：

1. 搜索和过滤主要在前端本地完成
2. 页面实际只拉取任务列表，然后前端筛选
3. 路由 query 会同步保存筛选状态和当前选中任务

## 6.3 概览指标卡

当前展示的指标：

1. 进行中
2. 已完成
3. 需要处理（失败）

## 6.4 空态

有两类空态：

1. 整个系统还没有任务
2. 当前筛选条件下没有任务

空态区包含：

1. 说明文案
2. 创建任务按钮或清空筛选按钮
3. 快速入口
4. 几个能力标签

## 6.5 任务分组

任务按生命周期分为 5 个分组：

1. 处理中
2. 已暂停
3. 排队中
4. 已完成
5. 失败

每组支持折叠 / 展开。

## 6.6 任务卡片 / 任务行

两种视图本质上承载相同信息。

每个任务最少包含：

1. 标题
2. 来源文件名或默认“文本生成任务”
3. 状态
4. 画幅
5. 进度
6. 时长范围
7. 重试次数
8. 效果评分
9. Seed
10. 更新时间
11. 标签：时间轴字幕 / 文本输入 / 需要处理 / 可继续生成 / 可查看结果

每个任务支持的动作取决于状态：

1. 暂停：`PENDING / ANALYZING / PLANNING`
2. 继续生成：`PAUSED`
3. 终止：`PENDING / ANALYZING / PLANNING / RENDERING`
4. 失败重试：`FAILED`
5. 删除：非运行中任务
6. 展开详情

## 6.7 任务详情侧

点击任务后，下方会出现详情区。当前详情区不是单独页面，而是同页展开。

### A. 详情头部

展示：

1. 标题
2. 任务 ID
3. 当前阶段
4. 进度
5. 刷新按钮
6. 根据状态显示暂停 / 继续 / 终止

### B. 阶段时间线

当前固定为 4 个阶段：

1. 素材分析
2. 任务编排
3. 视频生成
4. 任务完成

每个阶段状态包括：

1. 等待
2. 进行中
3. 已暂停
4. 已完成
5. 失败

### C. 结果概览

当前展示的是摘要，不是完整产物浏览器。

字段：

1. 已生成结果数量
2. 比例
3. 当前 Worker
4. Join 进度
5. 效果评分
6. 任务 Seed

### D. 效果评分

功能：

1. 1-5 分打分
2. 备注输入
3. 保存评分

说明：

1. 完成任务和失败任务都允许评分
2. 评分会被生成页的“高分 Seed 复用”模块再次使用

### E. 执行监控

如果后端返回 `monitoring`，当前会展示：

1. 当前阶段
2. Attempt 状态
3. 恢复阶段
4. 恢复镜头
5. 计划镜头数
6. 已生成镜头数
7. 连续完成镜头
8. 最新片段

### F. 产物目录

如果后端返回 `artifactDirectories`，当前会展示目录级信息：

1. Storage 根目录
2. 任务基目录
3. 运行目录
4. 拼接目录
5. 脚本文件名
6. 首帧命名规则
7. 尾帧命名规则
8. 片段命名规则
9. 拼接命名规则

说明：这是偏运维/调试信息，不是普通用户结果浏览。

### G. 创建参数

当前详情页会完整回看创建任务时的关键参数：

1. 文本模型
2. 视觉模型
3. 关键帧模型
4. 视频模型
5. 清晰度 / 画幅
6. 输出数量
7. 请求时长
8. 生效时长
9. 任务 Seed
10. 是否提前停止视频生成
11. 文本输入摘要
12. 画幅比例
13. 创意提示词
14. 文本输入摘要文本片段

### H. 最近阶段日志

显示最近 8 条日志，字段包括：

1. 时间
2. 阶段
3. 级别
4. 消息

## 10. 对 UI 生成 AI 的约束建议

如果把本项目交给别的 AI 重新生成 UI，建议明确以下约束，避免它发散：

1. 不要新增业务流程，只重做现有流程的视觉与交互组织。
2. 当前主流程只有 3 个页面：首页、生成页、任务页。
3. 生成页必须保留“模型链路 + 文本输入 + AI 生成提示词 + 实时进度 + 高分 Seed 复用”。
4. 任务页必须保留“筛选、分组、两种视图、详情区、任务动作、评分”。
5. 当前任务详情以“监控和参数回看”为主，不是媒体资产库页面。
6. 后台管理台虽然代码已实现，但当前不属于主导航的上线范围，除非明确要求一起重做。
7. 开发者模式是实际功能，不要删掉，只是可以弱化视觉权重。

## 11. 可直接给 UI 生成 AI 的摘要

你要设计的是一个“AI 文生视频工作台”的前端，当前真实主流程只有 3 个页面：

1. 首页：品牌展示 + 两个入口按钮。
2. 生成页：左侧配置任务参数和模型链路，中间/右侧填写创意提示词与小说正文，支持上传 TXT、AI 生成提示词、复用高分 Seed，提交后右侧实时显示任务进度和最近 trace。
3. 任务页：查看全部任务，支持搜索、状态筛选、排序、卡片/列表切换、按生命周期分组；点击任务后在同页展开详情，查看阶段时间线、结果摘要、执行监控、创建参数、评分和最近日志，并支持暂停、继续、终止、重试、删除。

业务重点不是复杂素材编辑，而是“配置任务 -> 提交任务 -> 追踪任务 -> 复盘任务 -> 复用 Seed”这条链路。
